Case Study: NSWC Corona

← Back to Home

The Challenge

Identified systemic data integrity failures due to the use of hundreds of disparate SharePoint lists as databases for tracking critical project metrics. This resulted in massive data duplication, lack of visibility for leadership, and significant operational overhead across the command.

My Architectural Solution

Formulated and successfully advocated for a strategic shift from SharePoint to Microsoft Dataverse. This involved presenting the technical and business case to senior leadership (PEO Digital) to procure and sanction a dedicated Dataverse environment.

  • Data Modeling: Designed a fully normalized data model in Dataverse, creating relational tables and establishing authoritative data relationships.
  • Application Design: Architected and built a Model-Driven Power App as the primary user interface, providing a unified platform for data entry and reporting.
  • Security & Governance: Implemented a granular security model using Dataverse's role-based and row-level security to ensure least-privilege access to sensitive project data.
Microsoft Dataverse Power Apps Power BI DoD IL5 (FlankSpeed)

The Outcome

The resulting solution consolidated hundreds of workflows, eliminated data duplication, and reduced departmental overhead related to data management by an estimated 80%.

This project established a single source of truth for mission-critical project data and provided leadership with real-time, accurate visualization of command-level metrics.
